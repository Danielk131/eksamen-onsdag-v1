Accessibility and why it matters

To design and code websites in the way that people with disabilities can use them, is one of the main goals when making a website accessible.

Managing the workload

We had accessibility in mind from the start, and considered this when choosing the layout and color-scheme for our website. Sometimes it is difficult to choose colors, because there is so many of them. To make website more accessible, we checked colors at Color Contrast Accessibility Validator and we went with light color-scheme with high contrasts. We kept the layout clear, with easy navigation of the website. You can also easily navigate the website by using only the Tab and Enter keys, which is a very important aspect of web accessibility.

We included the main page regions to our website, such as header, navigation, main content and footer, so that it would be easier for web browsers to identify them.
